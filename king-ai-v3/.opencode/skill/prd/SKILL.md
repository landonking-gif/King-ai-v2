# PRD Generation Skill

When asked to generate a PRD, follow this structure:

1. **Title**: Create a clear, descriptive title for the feature/product.

2. **Overview**: Provide a high-level description of what the feature/product does.

3. **Goals and Objectives**: List the business goals and user objectives.

4. **Target Audience**: Describe who the users are.

5. **User Stories**: Break down into user stories with acceptance criteria.

6. **Technical Requirements**: Any technical constraints or requirements.

7. **Timeline**: Estimated timeline if applicable.

8. **Success Metrics**: How to measure success.

Output in markdown format.
